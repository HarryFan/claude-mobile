# Sales Kit

為「B2B 科技品牌敘事型 scroll-driven landing page」服務打造的完整銷售資產。
定位、語氣、價格區間以 [one-pager.md](one-pager.md) 為唯一事實來源（single source of truth）——其他文件與它衝突時，改其他文件。

## 銷售流程與對應資產

```
認知          → 觸及              → 會談                    → 提案                  → 成交後
AURA X 作品   → outreach-         → discovery-call-         → proposal-             → (交付流程，
+ 宣傳影片      templates.md        script.md                 template.md             不在本 kit)
                ↑ 附 case-study     ↑ 會談前必讀 faq-        ↑ 會談後 24h 內
                  或 one-pager       objections.md             承諾送達日期
```

## 文件索引

| 文件 | 性質 | 何時用 |
| --- | --- | --- |
| [one-pager.md](one-pager.md) | 對外 | 服務總覽。首次接觸後想深入了解的 prospect；也是所有文件的定位基準 |
| [case-study.md](case-study.md) | 對外 | AURA X 案例。outreach 附件、會談前暖場、提案附錄 |
| [outreach-templates.md](outreach-templates.md) | 內部 | 五套開發模板：cold email ×2、DM、溫回覆、follow-up |
| [discovery-call-script.md](discovery-call-script.md) | 內部 | 20 分鐘工作會談腳本。所有 outreach 的 CTA 都指向這裡 |
| [faq-objections.md](faq-objections.md) | 內部 | 15 個常見疑慮回應框架。會談與提案階段的底稿 |
| [proposal-template.md](proposal-template.md) | 對外(模板) | 正式提案。會談收斂到 A 之後，依 Internal Notes 補齊送出 |

## 使用紀律

- **對外文件不直接暴露內部文件**：faq-objections、discovery-script、outreach-templates 是底稿，不寄給客戶
- **一次觸及一個 CTA**：全流程只推「20 分鐘工作會談」，不同時推 portfolio、報價、訂閱
- **價格只在兩個地方出現**：one-pager 的參考區間、proposal 的正式報價。outreach 與會談不報價
- **AURA X 每份文件最多引用一次**：proof 是敲門磚，不是全部內容
- **提案前必填** proposal-template.md 的 Internal Notes 五項
