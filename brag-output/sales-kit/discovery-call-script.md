# Discovery Call Script（20 分鐘工作會談）

內部文件。目標不是成交，是判斷：這個案子值不值得提案、落在哪個區間、對方是不是對的客戶。
會談本身就是產品的第一次體驗——節奏、提問品質、克制度，都在示範我們怎麼工作。

## 會談結構（20 分鐘）

| 段落 | 時間 | 目的 |
| --- | --- | --- |
| 開場定調 | 2 min | 設定議程，確立這是工作會談不是銷售電話 |
| 商業脈絡 | 6 min | 挖出頁面背後真正的商業目的 |
| 現況診斷 | 6 min | 對方眼中的問題 vs 我們看到的問題 |
| 方向示範 | 4 min | 給 1-2 個具體判斷，展示工作方式 |
| 下一步 | 2 min | 明確收斂：提案、再談、或不合適 |

---

## 1. 開場定調（2 min）

> 「今天 20 分鐘，我想聚焦三件事：這個頁面真正需要傳達什麼、現在的網站哪裡低估了產品、以及什麼程度的互動對你們是策略上合理的。最後如果方向對，我再說明合作怎麼進行。可以嗎？」

- 議程先講完，主導權立住
- 不寒暄超過一分鐘，克制本身就是品牌展示

## 2. 商業脈絡（6 min）

核心問題（依序，追問比換題重要）：

1. **「這個頁面要服務什麼商業時刻？」**（launch / 募資 / 改版 / enterprise 信任）
2. **「頁面的訪客是誰？他們看完之後你希望發生什麼？」**（詢問 / demo 預約 / 投資人信任 / 招聘）
3. **「如果這個頁面做對了，三個月後你怎麼知道？」**（詢問品質、轉換、品牌回饋）
4. **「時間壓力是什麼？」**（launch 日期已定 → 範圍倒推；沒有日期 → 決策節奏風險）

紅旗（記下，影響報價與是否接）：
- 說不出頁面的商業目的，只說「想要酷一點」
- 決策者不在場且無法接觸
- 期待「像 Apple 那樣」但預算在 Starter 以下
- 素材、文案、產品事實全部「之後再說」

## 3. 現況診斷（6 min）

1. **「你自己覺得現在的頁面最大的問題是什麼？」**（先聽對方的版本）
2. **「這個頁面上線以來，最常收到什麼樣的詢問？品質如何？」**
3. **「有沒有哪一段你們花很多力氣做，但好像沒人在乎？」**

然後給我們的診斷——會談前必須看過對方網站，準備好 2-3 個具體觀察：
- Hero 訊息是否回答「為什麼重要」而不只是「是什麼」
- 價值排序是否反映決策者關心的順序
- 互動 / 視覺是否有敘事功能，還是裝飾

> 「我會談前看了你們的頁面。如果只能改一件事，我會先動 `[具體觀察]`，因為 `[理由]`。」

## 4. 方向示範（4 min）

不做完整提案，給 1-2 個判斷示範工作方式：

- 敘事層：「你們的產品適合 `[敘事切角]`，因為 `[產品特性]`」
- 互動層：「以你們的產品，`[某種互動]` 有敘事功能；`[另一種]` 就只是裝飾，我不會建議」

引用 AURA X 一次就好：
> 「AURA X 想證明的就是這件事——高技術密度的體驗可以維持清晰跟商業導向。你們的案子適用同一個原則，但長相會完全不同。」

## 5. 下一步（2 min）

三種收斂，明確選一種：

**A. 方向對、預算對 → 提案**
> 「我聽到的需求落在 `[Starter / Core / Signature]` 區間。我 `[X 天]` 內給你一份正式提案，包含範圍、時程跟報價。提案會需要你確認 `[素材 / 決策窗口 / launch 日期]`。」

**B. 方向對、資訊不足 → 再一次會談**
> 「在報價之前我需要 `[決策者參與 / 素材盤點 / 產品 demo]`。我們約 `[時間]` 再談一次，那次之後我就能給提案。」

**C. 不合適 → 誠實收尾**
> 「聽起來你們現階段更適合 `[模板工具 / 先做文案 / 等產品定位穩定]`。現在做客製頁面對你們不划算。之後到了 `[時機]` 再回來找我。」

C 選項照講。拒絕不合適的案子是定位的一部分，也是轉介紹的來源。

---

## 會談後 24 小時內

- 寄 recap email：三點會談結論 + 明確下一步 + 時間承諾
- 內部記錄：區間判斷、紅旗、決策鏈、素材狀態（填入 proposal-template.md 的 Internal Notes）
- 走 A 的案子：提案在承諾日期前送達，逾期等於示範我們會逾期
